# DESIGN.md

## Mod Summary
- **Mod Name:** LiveStudioInteractiveMod
- **Description:** A GTA5 Story Mode LIVE Studio mod that reacts to chat and gift events. Viewers sending "ping" in chat will give the player +25 armor and show a notification. Receiving a "rose" gift spawns an invincible police car near the player.
- **Author:** Generated by Workflow
- **Version:** 1.0
- **Target:** GTA5 Story Mode only, ScriptHookVDotNet v3, LIVE Studio template.

## Interaction Map
### Chat Triggers
| Trigger Type | Keyword / Command | Condition | RepeatEnd Behavior | Cooldown (s) | Game Effect |
|--------------|-------------------|-----------|--------------------|--------------|-------------|
| Chat         | ping              | `ChatEvent.Content` equals `"ping"` (case‑insensitive) | N/A                | 10           | Show notification `"PING! +25 Armor"` and add 25 armor to the player Ped. |

### Gift Triggers
| Trigger Type | Gift ID | Condition | RepeatEnd Behavior | Cooldown (s) | Game Effect |
|--------------|---------|-----------|--------------------|--------------|-------------|
| Gift         | rose    | `GiftEvent.GiftId` equals `"rose"` **and** `GiftEvent.RepeatEnd == true` | Trigger **only** when `RepeatEnd` is `true` (final unit of a combo gift) | 10           | Attempt to spawn an invincible police car (`Police`) on a nearby street. Show notification `"Rose gift: Invincible Police Car!"`. Car is added to a `List<Vehicle>` for later cleanup. If no valid street position is found, show fallback notification `"Rose gift: Couldn't find a spot for the police car."` |

## Gameplay Rules
- All game‑modifying calls are dispatched via `MainThreadDispatcher.Enqueue` to ensure thread safety.
- **Chat (ping):**
  - Check if player Ped exists and is alive. If not, ignore.
  - `GTA.UI.Notification.Show("PING! +25 Armor")` for user feedback.
  - Call `PED.ADD_ARMOUR_TO_PED(PlayerPed, 25)`. Armour cap is handled natively; if already at maximum, no effect.
- **Gift (rose):**
  - Trigger only when `GiftEvent.RepeatEnd == true` (so combos of multiple roses generate exactly one car per combo, not one per unit).
  - Check if player Ped exists and is alive. If not, ignore.
  - Attempt to find a safe street position near the player using `World.GetNextPositionOnStreet(player.Position.Around(2.0f))`. If successful:
    - Create vehicle: `World.CreateVehicle(VehicleHash.Police, streetPos, heading, VehicleLockStatus.None)`.
    - Immediately call `SET_ENTITY_INVINCIBLE(vehicle, true)` to make it invincible.
    - Add vehicle to `modSpawnedVehicles` list for proper cleanup.
    - Show `GTA.UI.Notification.Show("Rose gift: Invincible Police Car!")`.
  - If no valid street position is found, show `GTA.UI.Notification.Show("Rose gift: Couldn't find a spot for the police car.")` and do not spawn a vehicle.
- **Cooldowns:**
  - Separate cooldown timers for chat and gift triggers, both 10 seconds.
  - Each cooldown uses `Game.GameTime` timestamp saved in a dictionary/field. Only process the event if current time - lastTriggerTime >= 10,000 ms.
  - Cooldown applies per trigger type globally; a new event of the same type within the cooldown is silently ignored.

## Game Effects
- **Notification Feedback:** All user‑visible messages use `GTA.UI.Notification.Show` (a banner notification that fades out after a few seconds). This fulfills the “display subtitle” requirement in a simple, reliable way.
- **Armor Addition:** Native `ADD_ARMOUR_TO_PED` instantly adds 25 points. No limit checking beyond the native cap.
- **Vehicle Spawn:** A `Police` vehicle is created at a valid on‑street position offset from the player. The vehicle is flagged invincible and cannot be destroyed under normal Story Mode play. It remains in the world until deleted by script cleanup or manual removal.
- **Cleanup:** On mod stop or script abortion, all vehicles stored in `List<Vehicle>` are deleted with `vehicle.Delete()` and the list is cleared. `LiveStudioClient.Dispose()` is called to detach event handlers and release resources.

## Edge Cases & Boundaries
- **Player unavailable:** If the player Ped is `null` or dead, both triggers are silently skipped (no effect).
- **Multiple combos in rapid succession:** Cooldown prevents spam; only the first completed combo triggers a car, subsequent combos within the cooldown window are ignored.
- **Vehicle spawn failure:** If `World.GetNextPositionOnStreet` returns `false`, no vehicle is spawned and a fallback notification is shown. The cooldown still applies to prevent immediate re‑triggering.
- **Invincible vehicle persistence:** The car stays invincible forever. No durability or time limit is implemented. If the game world is overloaded with vehicles, manual cleanup is left to the script (which clears them on mod unload). In long sessions, this may accumulate many vehicles unless the streamer manually deletes them.
- **Chat casing:** The keyword `ping` is matched case‑insensitively (e.g., `"PING"`, `"Ping"` all work).
- **Non‑LIVE Studio environment:** If the template is not connected to a live event source, the mod simply never fires. Keyboard fallback commands (e.g., Debug keys) are not included in this design, keeping the mod pure LIVE Studio. (Implementation may add optional debug keys later.)
- **GTA Online:** No Online features or references. The mod only functions in Story Mode.

## Design Notes
- This design assumes a fixed LIVE Studio template that provides `HandleEvent(LiveStudioEvent evt)` and event objects `ChatEvent`, `GiftEvent`. The mod will override `HandleEvent` and dispatch to appropriate methods.
- The mod’s state is kept minimal: two cooldown timestamps and a `List<Vehicle>` for spawned cars.
- For maximum compatibility, all game interactions run through `MainThreadDispatcher.Enqueue` even if the template already marshals events on the main thread. This ensures safety.
- Gift combos are handled by listening to `RepeatEnd == true` to avoid spawning many cars from one combo. This is the expected interaction for many streamer events. If the streamer prefers a car per unit, the design can be switched to ignore `RepeatEnd` and use a short cooldown instead.
- All code‑facing names (class, methods, variables), gift IDs, and in‑game text are in English.
- The `List<Vehicle>` is used for cleanup; the design avoids `new Vehicle(handle)` outside the scope of the spawned entity handle – vehicles are created via `World.CreateVehicle` which returns a `Vehicle` object that should be stored and later disposed with `Delete()`.
